# Product

## Register

brand

## Users

A developer with ~5 years of JS experience who needs to strengthen their core understanding for interviews. They know how to build things but struggle to articulate internals — closures, the event loop, prototypal inheritance, scope chains, `this` binding. They're preparing for technical interviews at companies that probe deep JS knowledge.

## Product Purpose

A structured, opinionated JavaScript learning hub that bridges the gap between "I can write JS" and "I can explain JS at an interview." Every topic goes beyond surface syntax into how it actually works under the hood. The landing page is the central navigation hub — not a marketing page, but a confident, well-organized table of contents that makes you want to dive in.

## Brand Personality

**Technical, authoritative, no-nonsense.** Three words: *precise, confident, direct.*

This is not a beginner tutorial site. The voice is a senior engineer who writes clean explanations without hand-holding. Think: well-edited Stack Overflow answers compiled into a structured reference, not a "hello world" tutorial series.

## Anti-references

- No generic documentation templates (Docusaurus default, JSDoc beige)
- No AI-generated landing page aesthetics (glassmorphism, gradient text, identical card grids, hero metrics)
- No marketing-heavy SaaS landing pages (CTA spam, glossy stock photos, testimonial carousels)
- No warm/crafty/tutorial-playground vibes (playful illustrations, emoji-heavy headings, gamification)
- No editorial-magazine aesthetics (display serif italics, drop caps, broadsheet grids) — this is technical, not editorial

## Design Principles

1. **Substance over style.** The content is the product. Design supports readability and navigation, never competes with it.
2. **Developer-native.** The site should feel like it was built by a developer, for developers. Terminal-influenced dark mode, monospace accents where they belong (code, labels, technical terms), not as costume.
3. **Structured depth.** Every section is a clear learning path. Progression from fundamentals to internals to interview patterns. The landing page maps this structure visually.
4. **Dark-first with light option.** The default experience is a dark theme optimized for focused reading. A light toggle exists for daytime or preference.
5. **Show, don't decorate.** Code examples and diagrams carry the visual weight. No decorative SVGs or placeholder imagery. When in doubt, less.
